## 当前编译问题分析

### 剩余错误类型统计
```
类型错误 (E0308): ~45 处
数据库反序列化模式: ~10 处  
```

### 核心问题
所有错误都集中在**数据库 Repository 层**的错误处理模式不统一：

```rust
// ❌ 错误模式 1 - 闭包中无法使用 ?
Ok(row.map(|r| Account {
    field: parse()?  // 无法在 map 闭包中使用 ?
}))

// ✅ 正确模式 - 使用 match
match row {
    Some(r) => Ok(Some(Account {
        field: parse()?,  // 可以在外层函数中使用 ?
    })),
    None => Ok(None),
}
```
